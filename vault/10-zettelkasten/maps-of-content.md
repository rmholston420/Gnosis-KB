# Maps of Content

A Map of Content (MOC) is an index note that gathers related notes into a navigable overview. Unlike a folder, a MOC is itself a note — it can be linked, annotated, and evolved over time. MOCs are the primary navigation mechanism in a mature Zettelkasten.

## When to Create a MOC

Create a MOC when you notice:
- A cluster of 5+ notes that share a theme
- You keep forgetting which notes cover a topic
- A topic has grown complex enough to deserve an overview

## Anatomy of a MOC

```
## Overview
Brief description of the territory.

## Core Notes
- [[Key Concept A]]
- [[Key Concept B]]

## Entry Points by Sub-Topic
### Sub-topic 1
- [[Note]]

## Open Questions
- What remains unresolved?
```

## MOC vs. Tag vs. Folder

| Mechanism | Strength | Weakness |
|-----------|----------|----------|
| Folder    | Simple, visible | Rigid, one location |
| Tag       | Multi-dimensional | No narrative |
| MOC       | Narrative, linkable | Requires curation |

Use all three: folders for coarse structure, tags for cross-cutting dimensions, MOCs for navigable overviews of dense topic clusters.
